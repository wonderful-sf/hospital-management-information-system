# 03-住院管理

## 功能名

住院管理

## 需求描述

覆盖住院完整流程：住院建档 → 分配病床 → 预缴费用 → 每日住院记录与扣费 → 余额不足暂停治疗 → 出院结算，由管理员、医生、病人三方协作完成。

## 概述

住院是系统另一核心业务。病人经门诊诊断后由管理员办理住院建档、分配病床；病人预缴费用；主治医生每日巡诊填写住院记录并生成当日费用；预缴余额不足时自动暂停治疗；出院时多退少补，生成最终账单。

## 相关页面设计

### 1. 住院档案管理页（管理员端）

**表格列**：住院号、病人、科室、主治医生、病床、入院时间、出院时间、预缴余额（元）、状态标签

**状态标签**：`ACTIVE`(绿)、`DISCHARGED`(灰)、`SUSPENDED`(橙)

**筛选**：病人姓名、科室、状态、入院日期范围

**新增住院档案弹窗**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 病人 | select | * | 从 patients 加载 |
| 科室 | select | * | 选择后加载该科室医生和可用病床 |
| 主治医生 | select | * | 该科室下 ACTIVE 医生 |
| 病床 | select | | 可选，该科室下 AVAILABLE 且未被占用的病床 |
| 预缴金额（元） | number | * | 最低 500 元 |

**编辑**（仅 ACTIVE 状态）：可更换病床、更换主治医生。

**出院按钮**（仅 ACTIVE 状态）：触发出院结算流程。

### 2. 预缴管理页（病人端）

**上方**：当前预缴余额大号数字显示（余额为负时红色高亮）

**表格**：流水号、金额（元）、类型标签、余额变动后（元）、时间、备注

**类型标签**：`DEPOSIT`(绿)、`DEDUCTION`(红)、`REFUND`(蓝)

**充值入口**：输入金额 → 确认 → 余额增加

**余额不足警告**：余额 < 0 时红色提示"预缴余额不足，请及时充值以继续治疗"。

### 3. 住院记录管理页（医生端）

**布局**：左侧负责病人列表 + 右侧住院记录表格和表单

**右侧表格**：该病人的每日住院记录（日期、病情摘要、当日费用）

| 字段 | 类型 | 必填 | 说明 |

| 用法 | text | | |

**预缴充值入口**：ACTIVE 状态的档案旁显示"充值"按钮。

## 用户旅程

### 住院建档
管理员登录 → 住院管理 → 住院档案 → 新增 → 选择病人李四 → 科室"内科" → 系统列出内科主治医生和可用病床 → 选择张医生、301-1 床 → 预缴 2000 元 → 提交 → 住院号 AN202605150001 → 病床 OCCUPIED → 余额 2000。

### 每日巡诊
医生登录 → 住院记录 → 选择李四(AN202605150001) → 新增记录 → 日期 2026-05-15 → 病情"体温正常，咳嗽减轻" → 方案"继续抗感染" → 药品"头孢"×2 → 治疗"雾化吸入"×1 → 当日费用 120 元 → 提交 → 余额 = 1880。

### 余额不足暂停
连续扣费 → 余额 = -50 → 自动 SUSPENDED → 病人端红色提示 → 充值 1000 元 → 余额 = 950 → 状态恢复 ACTIVE。

### 出院结算
管理员 → 住院档案 → 出院 → 系统汇总总费用（床位费 + 各日记录费用）→ 结余多退少补 → 病床 AVAILABLE → 档案 DISCHARGED → 生成最终账单。

## 用户故事

1. 作为管理员，办理住院时系统应自动校验病床可用性，避免重复分配。
2. 作为病人，我希望随时查看预缴余额和消费明细，费用透明。
3. 作为医生，填写每日记录时系统应自动计算当日费用。
4. 作为管理员，办理出院时系统应自动汇总全部费用并多退少补。
5. 作为病人，余额不足时希望收到明确提醒并及时充值。

## 实现逻辑

### 住院建档（同一事务）
```
a. 校验 bed 未被其他 ACTIVE 住院占用
b. INSERT admissions (admission_no, status='ACTIVE', prepaid_balance=0)
c. IF bed_id IS NOT NULL:
     UPDATE beds SET status='OCCUPIED'
d. INSERT prepaid_records (type='DEPOSIT', amount, balance_after=amount)
e. UPDATE admissions SET prepaid_balance = amount
```

### 每日住院记录（同一事务）
```
a. 校验 admission status='ACTIVE'
b. 校验同一天无重复记录 (admission_id + record_date)
c. INSERT inpatient_records
d. FOR EACH 药品: INSERT items + UPDATE stock
e. FOR EACH 治疗: INSERT items
f. 当日总费用 = treatment_fee + SUM(药品) + SUM(治疗)
g. INSERT prepaid_records (type='DEDUCTION', amount, balance_after)
h. UPDATE admissions SET prepaid_balance -= 当日总费用
i. IF prepaid_balance < 0: UPDATE admissions SET status='SUSPENDED'
```

### 出院结算（同一事务）
```
a. 汇总总费用 = 床位费 + 所有记录费用
b. IF prepaid_balance > 0: INSERT prepaid_records (type='REFUND')
c. INSERT bills (source_type='INPATIENT', total_amount, status='PAID')
d. INSERT bill_items (BED + MEDICINE + TREATMENT)
e. UPDATE beds SET status='AVAILABLE'
f. UPDATE admissions SET status='DISCHARGED', discharged_at=NOW()
```

### 预缴充值
```
a. 校验 admission status IN ('ACTIVE','SUSPENDED')
b. INSERT prepaid_records (type='DEPOSIT', balance_after=新余额)
c. UPDATE admissions SET prepaid_balance += amount
d. IF status='SUSPENDED' AND prepaid_balance >= 0:
     UPDATE admissions SET status='ACTIVE'
```

## 功能细节描述

- 住院号格式 `AD{yyyyMMdd}{4位序号}`
- 同一 admission_id 同一天只能有一条住院记录（DB UNIQUE 约束）
- 预缴最低 500 元
- 床位费 = wards.daily_charge × 住院天数
- 出院后档案和记录只读
- 医生仅看自己负责的病人
- prepaid_records 可追溯全部收支
- 病床可暂不分配，后续再分配

## 验收标准

- [ ] 管理员能办理住院建档、分配病床、办理出院
- [ ] 病人能查看预缴余额和充值
- [ ] 病人能查看住院档案和每日记录
- [ ] 医生能查看自己负责的住院病人并填写每日记录
- [ ] 每日记录提交后自动扣费更新余额
- [ ] 余额不足自动 SUSPENDED
- [ ] 充值后余额 ≥ 0 时恢复 ACTIVE
- [ ] 出院结算正确汇总并多退少补
- [ ] 病床出院后恢复 AVAILABLE
- [ ] 同一天不可重复提交住院记录
