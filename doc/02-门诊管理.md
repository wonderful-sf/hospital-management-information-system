# 02-门诊管理

## 功能名

门诊管理

## 需求描述

覆盖门诊完整流程：排班 → 挂号 → 接诊 → 开具处方，由管理员、医生、病人三方协作完成。门诊是系统核心业务流程。

## 概述

病人挂号选择科室和医生；医生按排班接诊后填写诊断并开具处方；处方关联药品明细和诊疗费，支持纯诊疗（不开药）；处方提交后自动生成账单；病人缴费后处方状态变为 PAID。门诊不涉及药房发药流程。

### 2. 排班管理页（管理员）

**表格列**：医生、科室、排班类型、开始时间、结束时间、诊室、创建时间

**筛选**：科室、排班类型、日期范围

**新增排班弹窗**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 医生 | select | * | 只选 ACTIVE 状态的医生 |
| 科室 | select | * | 自动带出医生所属科室 |
| 排班类型 | select | * | OUTPATIENT / INPATIENT_ROUND |
| 日期 | date | * | |
| 开始时间 | time | * | 精确到分钟 |
| 结束时间 | time | * | 必须晚于开始时间 |
| 诊室 | text | | 如"门诊 201 室" |

**时间冲突校验**：同一医生同一天内新排班与已有排班时间不可重叠。

**编辑/删除**：排班开始时间已过的记录只读。

### 3. 挂号管理页（病人端）

**表格列**：病案号、医生、科室、初诊/复诊标签、挂号时间、状态标签

**状态标签**：`REGISTERED`(蓝)、`VISITED`(绿)、`CANCELLED`(灰)

**筛选**：状态、日期范围

**新增挂号弹窗**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 科室 | select | * | 加载有排班的科室 |
| 医生 | select | * | 加载该科室下当前排班为 OUTPATIENT 且时间段未过期的医生 |
| 就诊类型 | select | * | FIRST（初诊）/ FOLLOW_UP（复诊） |
| 排班时段 | select | * | 显示该医生的可用排班时段 |

**取消挂号**：仅 `REGISTERED` 状态可取消。

### 4. 挂号管理页（管理员端）

**表格列**：全部挂号记录（病人、医生、科室、就诊类型、挂号时间、状态）

**筛选**：病人姓名、医生姓名、科室、状态、日期范围

### 5. 接诊管理页（医生端）

**布局**：左侧待接诊列表 + 右侧接诊表单

**待接诊列表**：今日 REGISTERED 状态的挂号，按挂号时间排序

**接诊表单**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 病人 | text | | 只读 |
| 挂号信息 | text | | 只读 |
| 症状描述 | textarea | * | |
| 诊断意见 | textarea | * | |

**提交后**：挂号状态 → VISITED，生成 outpatient_visit 记录。

**已接诊列表**：历史接诊记录，按日期筛选。

### 6. 处方管理页（医生端）

**表格列**：病人、诊断摘要、诊疗费（元）、药品费（元）、总金额（元）、状态标签、创建时间

**状态标签**：`UNPAID`(红)、`PAID`(绿)

**筛选**：病人姓名、日期范围、状态

**新增处方**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 就诊记录 | select | * | 选择已接诊但尚未开处方的 outpatient_visit |
| 诊疗费（元） | number | * | 默认带入 doctor_titles.consultation_fee，可手动调整 |

**药品明细（可多行添加）**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 药品 | search-select | * | 模糊搜索，只显示 ACTIVE 且库存>0 的药品 |
| 数量 | number | * | 正整数 |
| 用法 | text | | 如"每日 3 次，每次 1 粒" |
| 小计 | number | | 自动计算 = 单价 × 数量 |

**允许不开药品**：药品明细可为空（纯诊疗），此时 medicine_amount=0，total_amount=consultation_fee。

**提交后（同一事务）**：
- INSERT prescriptions + prescription_items
- UPDATE medicines.stock_quantity -= 数量
- INSERT bills (source_type='OUTPATIENT', status='UNPAID') + bill_items

### 7. 我的处方页（病人端）

**表格列**：诊断、医生、诊疗费、药品费、总金额、状态标签、创建时间

**展开行**：点击行展开药品明细（药品名、数量、单价、小计、用法）

**缴费入口**：UNPAID 处方旁显示"去缴费"按钮。

## 用户旅程

### 病人挂号
病人登录 → 门诊管理 → 挂号管理 → 新增挂号 → 选择科室"内科" → 系统列出内科有门诊排班的医生和时间 → 选择张医生、5月15日 09:00-12:00 → 选择"初诊" → 提交 → 挂号列表新增 REGISTERED 记录。

### 医生接诊开方
医生登录 → 接诊管理 → 待接诊列表看到张三 → 点击"接诊" → 填写症状"发热咳嗽 3 天"和诊断"上呼吸道感染" → 提交 → 挂号变 VISITED → 进入处方管理 → 新增处方 → 选择该就诊记录 → 诊疗费 30 元 → 添加"阿莫西林"×2、"布洛芬"×1 → 总金额 86 元 → 提交 → 库存扣减 + 生成账单。

### 纯诊疗（不开药）
医生接诊后 → 新增处方 → 选择就诊记录 → 诊疗费 30 元 → 不添加药品 → 提交 → medicine_amount=0, total_amount=30 → 生成账单（仅含诊疗费明细）。

## 用户故事

1. 作为病人，我希望挂号时只看到有门诊排班的医生，避免挂空号。
2. 作为医生，我希望接诊列表按挂号时间排序，优先处理先到的病人。
3. 作为医生，开处方时系统应提示药品库存不足，避免开出无效处方。
4. 作为管理员，我希望查看全部挂号记录，了解门诊流量。
5. 作为病人，我希望不开药时也能正常就诊并支付诊疗费。

## 实现逻辑

### 挂号创建
```
a. 校验 schedule_id 存在且 start_time > NOW()
b. INSERT registrations (status='REGISTERED')
```

### 接诊
```
a. 校验挂号 status='REGISTERED'
b. UPDATE registrations SET status='VISITED'
c. INSERT outpatient_visits (symptom_description, diagnosis)
```

### 处方创建（同一事务）
```
a. 校验 outpatient_visit 尚未关联处方
b. 校验每种药品 status='ACTIVE' AND stock_quantity >= quantity
c. INSERT prescriptions (consultation_fee, medicine_amount, total_amount)
d. FOR EACH 药品:
     INSERT prescription_items
     UPDATE medicines SET stock_quantity -= quantity
e. INSERT bills (source_type='OUTPATIENT', source_id=prescription.id,
   total_amount, status='UNPAID')
f. INSERT bill_items (CONSULTATION 类型)
g. FOR EACH 药品:
     INSERT bill_items (MEDICINE 类型)
h. 任一步失败 → 事务回滚
```

## 功能细节描述

- 排班时间精确到分钟，同一天同一医生可创建多个不重叠时段
- 挂号后不做自动过期取消
- 处方一旦创建不可修改
- 诊疗费从 doctor_titles.consultation_fee 默认带入，医生可手动调整
- 药品选择支持模糊搜索，下拉显示药品名/规格/库存/单价
- 处方药品明细允许为空，此时 medicine_amount=0
- bill 的 source_type='OUTPATIENT'，source_id=prescription.id
- 排班管理页管理员增删改排班后，日历页实时刷新

## 验收标准

- [ ] 管理员能新增/编辑/删除排班，时间冲突时提示
- [ ] 排班日历正确展示三种角色的数据权限
- [ ] 病人能根据科室和排班挂号和取消挂号
- [ ] 医生能查看待接诊列表并填写诊断
- [ ] 医生能开具带药品的处方和纯诊疗处方
- [ ] 处方提交后自动扣减库存和生成账单
- [ ] 库存不足时处方创建失败并回滚
- [ ] 历史排班只读
- [ ] 已接诊的挂号不可取消
