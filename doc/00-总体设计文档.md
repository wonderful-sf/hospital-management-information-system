# 00-总体设计文档

## 1. 系统概述

### 1.1 项目背景

本项目基于《数据库系统原理与实践》综合设计实验指导书，使用 MySQL 数据库 + Spring Boot + MyBatis 实现医院管理信息系统。业务范围包括门诊治疗、住院治疗、药品信息维护、线上缴费与统计查询；不涉及医疗检查、手术治疗和药房发药业务流程。

### 1.2 系统目标

- 支持管理员维护科室、医生、病人、药品、病房、病床等基础数据。
- 支持医生查看排班、接诊病人、开具门诊处方和住院诊疗方案。
- 支持病人挂号、缴费、就诊、查询就诊记录和费用明细。
- 支持住院建档、病床分配、预缴费用、每日住院记录、余额不足暂停治疗、出院结算。
- 支持统计不同科室的排班情况、不同医生的工作量、病人的治疗情况。

### 1.3 角色定义

| 角色 | 标识 | 说明 |
|------|------|------|
| 管理员 | `ADMIN` | 维护基础资料、管理住院档案、查看全部数据和统计 |
| 医生 | `DOCTOR` | 查看排班、接诊病人、开具处方、填写住院记录、查看本人统计 |
| 病人 | `PATIENT` | 挂号、查看就诊记录和处方、缴费、预缴充值、查看住院档案 |

### 1.4 系统边界

- 包含：科室/医生/病人/药品/病房/病床/排班/挂号/门诊就诊/处方/住院档案/住院记录/账单/支付/预缴/统计
- 不包含：药房发药、医疗检查与检验、手术治疗、真实支付对接、医保结算
- 不包含：药品管理员角色（药品维护由管理员承担）、收费员角色（缴费由病人自行操作）

## 3. 数据库设计

### 3.1 设计原则

- 统一 `users` 表管理登录账号，通过 `role` 区分管理员、医生、病人。
- 课程实验中密码按明文存储，字段命名为 `password`。
- 不使用数据库物理外键，所有关联关系通过业务字段和应用层逻辑维护。
- 门诊和住院业务分开建模，但共享医生、病人、科室、药品等基础表。
- 费用统一由账单和账单明细表达，账单状态只区分未支付和已支付。
- 药品价格、医生诊疗费等费用信息在业务明细中保留快照。
- 支付和预缴通过独立表记录，支持资金流水追溯。

### 3.3 现有表调整

相对于原设计，以下表有字段变更：

**`users` 表**：新增 `permissions VARCHAR(500)` 列，存储 JSON 权限标识数组。

**`admissions` 表**：
- 新增 `prepaid_balance DECIMAL(10,2) NOT NULL DEFAULT 0`（预缴余额）
- 新增 `discharge_type ENUM('NORMAL','AGAINST_ADVICE','TRANSFERRED')`（出院类型）
- `bed_id` 改为 `BIGINT DEFAULT NULL`（入院时可能暂未分配）

**`bills` 表**：新增 `paid_at DATETIME`（支付时间）。

其余 15 张表结构不变。完整表结构见 `doc/表结构说明.md`。

### 4.3 数据权限规则

| 规则 | 说明 |
|------|------|
| 医生只看自己的排班 | `doctor_schedules` 按 `doctor_id` 过滤 |
| 医生只看接诊自己的挂号 | `registrations` 按 `doctor_id` 过滤 |
| 医生只看自己开具的处方 | `prescriptions` 按 `doctor_id` 过滤 |
| 医生只看自己负责的住院档案 | `admissions` 按 `attending_doctor_id` 过滤 |
| 病人只看自己的信息 | 所有业务表按 `patient_id` 过滤 |
| 管理员看全部 | 不做数据过滤 |

### 4.4 权限校验流程

1. 用户登录 → 校验 `username + password` → 检查 `status = ACTIVE`
2. 登录成功 → 将 `users` 信息 + `permissions` JSON 数组加载到 Session
3. 前端：页面加载时从 Session 读取 `permissions`，隐藏无权限的菜单和按钮
4. 后端：每个 Controller 方法通过拦截器校验对应权限标识
5. 后端：Service 层根据角色自动注入数据权限过滤条件

## 5. 公共组件与交互规范

### 5.1 统一后台布局

- **顶部栏**：左侧系统名，右侧当前用户名 + 退出按钮

### 5.2 通用组件

| 组件 | 说明 |
|------|------|
| 数据表格 | 支持关键词搜索、分页（默认 20 条/页） |
| 表单弹窗 | 新增/编辑操作弹出模态框，含取消和提交按钮 |
| 状态标签 | 不同状态对应不同颜色标签，详见各模块 |

- 表格搜索：输入关键词后触发后端搜索

- 金额显示：保留两位小数
- 日期显示：统一 `yyyy-MM-dd`，时间显示 `yyyy-MM-dd HH:mm`

## 7. 全局约束

1. 不使用数据库物理外键，一致性由应用层保证。
2. 密码明文存储（`users.password`），满足课程实验要求。
3. 不引入前端框架，纯 HTML + CSS + 原生 JS。
4. 费用字段使用 DECIMAL(10,2)，避免浮点精度问题。
5. 价格、诊疗费在业务明细表中保存快照，后续母表价格变动不回溯历史记录。
6. 所有编号由后端统一生成。
7. 数据库写操作使用事务，保证数据一致性。
