# 01-基础资料管理

## 功能名

基础资料管理

## 需求描述

管理员维护病人、科室、医生、药品、病房、病床等基础数据，是整个系统的数据底座。基础资料是门诊、住院、费用模块的前置依赖，所有业务模块引用这些基础数据。

## 概述

管理员登录后可对六类资料进行查询、新增、编辑、启用/停用。基础资料的完整性和准确性直接影响下游业务流程。删除操作需要校验下游引用，存在关联数据时禁止删除并提示。

## 用户故事

1. 作为管理员，我希望新增科室时系统自动校验名称唯一性，避免重复创建。
2. 作为管理员，我希望删除科室前系统提示关联的医生和病房数量，避免误删导致数据断裂。
3. 作为管理员，我希望新增医生和病人时系统自动创建登录账号，减少重复录入。
4. 作为管理员，我希望药品停用后新处方不可再选，但不影响历史处方数据。
5. 作为管理员，我希望病床被占用时不可删除，确保住院数据完整性。
6. 作为管理员，我希望病人有挂号/住院/账单记录时禁止删除，保证数据可追溯。

## 实现逻辑

### 新增医生（同一事务）
```
a. INSERT users (username=工号, password='123456', role='DOCTOR',
   permissions='["doctor:schedule:view","doctor:registration:view",
   "doctor:visit:manage","doctor:prescription:manage",
   "doctor:inpatient:view","doctor:inpatient:manage",
   "doctor:statistics:view"]', status='ACTIVE')
b. INSERT doctors (user_id=a.id, ...)
c. 若任一步失败则回滚
```

### 新增病人（同一事务）
```
a. INSERT users (username=病案号, password='123456', role='PATIENT',
   permissions='["patient:registration:manage","patient:registration:cancel",
   "patient:visit:view","patient:prescription:view",
   "patient:bill:view","patient:bill:pay",
   "patient:admission:view","patient:prepaid:manage",
   "patient:prepaid:view"]', status='ACTIVE')
b. INSERT patients (user_id=a.id, ...)
c. 若任一步失败则回滚
```

### 删除前置校验
```
科室删除前：
  SELECT COUNT(*) FROM doctors WHERE department_id=? → >0 禁止
  SELECT COUNT(*) FROM wards WHERE department_id=? → >0 禁止

医生删除前：
  SELECT COUNT(*) FROM doctor_schedules WHERE doctor_id=?
  SELECT COUNT(*) FROM registrations WHERE doctor_id=?
  SELECT COUNT(*) FROM outpatient_visits WHERE doctor_id=?
  SELECT COUNT(*) FROM prescriptions WHERE doctor_id=?
  SELECT COUNT(*) FROM admissions WHERE attending_doctor_id=?
  → 任一 >0 则禁止

病人删除前：
  SELECT COUNT(*) FROM registrations WHERE patient_id=?
  SELECT COUNT(*) FROM outpatient_visits WHERE patient_id=?
  SELECT COUNT(*) FROM prescriptions WHERE patient_id=?
  SELECT COUNT(*) FROM admissions WHERE patient_id=?
  SELECT COUNT(*) FROM bills WHERE patient_id=?
  → 任一 >0 则禁止

药品删除前：
  SELECT COUNT(*) FROM prescription_items WHERE medicine_id=?
  SELECT COUNT(*) FROM inpatient_record_items WHERE medicine_id=?
  → 任一 >0 则禁止

病房删除前：
  SELECT COUNT(*) FROM beds WHERE ward_id=? → >0 禁止

病床删除前：
  校验 status='AVAILABLE' 且未被 ACTIVE 住院占用
```

## 功能细节描述

- 科室名称、医生工号、病案号、药品编码、病房编号全局唯一（DB UNIQUE + 应用层校验）
- 医生工号和病案号由管理员手动输入，系统不自动生成
- 管理员不能 DISABLED 或删除自己的账号
- 所有删除操作弹出二次确认："确定删除 [名称]？此操作不可撤销。"
- 表格默认每页 20 条，支持分页
- 操作成功/失败均显示 Toast 提示
- 新增医生和病人默认密码 123456，首次登录后可由用户自行修改（二期）
- 药品库存修改记录当前值，不做历史追溯

## 验收标准

- [ ] 管理员能正常新增/编辑/删除病人、科室、医生、药品、病房、病床
- [ ] 科室名称/医生工号/病案号重复时新增失败并提示
- [ ] 有下游引用的记录删除时被拦截并提示关联数量
- [ ] 新增医生和病人时自动创建 users 账号，权限标识正确
- [ ] 药品 DISABLED 后处方药品选择中不出现
- [ ] 病床 OCCUPIED 时不可删除
- [ ] 病人有挂号/住院/账单时不可删除
- [ ] 表格搜索和分页正常工作
