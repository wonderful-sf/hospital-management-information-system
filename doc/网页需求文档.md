## 1. 页面建设目标

基于实验书、数据库设计文档、表结构说明和模拟数据，为医院信息管理系统制作课程展示型前端页面。前端先采用 HTML + CSS + JavaScript 静态页面实现，不依赖后端接口，后续可按本需求文档调整后重新生成页面。

## 2. 技术范围

- 页面位置：`src/main/resources/static/`
- 样式文件：`src/main/resources/static/css/index.css`
- 脚本文件：`src/main/resources/static/js/index.js`
- 技术栈：HTML、CSS、原生 JavaScript
- 不引入第三方前端框架
- 不实现真实登录、数据库读写或后端接口调用
- 页面数据暂用 `doc/seed.sql` 中的模拟数据静态展示

## 3. 页面清单

| 页面 | 文件名 | 主要用途 |
|---|---|---|
| 首页 | `index.html` | 系统介绍、角色入口、模块总览 |
| 管理员工作台 | `admin.html` | 基础资料维护展示 |
| 医生工作台 | `doctor.html` | 排班、接诊、处方、巡诊展示 |
| 病人中心 | `patient.html` | 挂号、就诊、处方、费用查询展示 |
| 住院管理 | `inpatient.html` | 住院档案、病床、住院记录展示 |
| 费用管理 | `billing.html` | 门诊和住院账单展示 |
| 统计分析 | `statistics.html` | 科室排班、医生工作量、治疗情况统计 |

## 4. 首页需求

首页用于课程展示和系统入口，应包含：

1. 系统名称：医院信息管理系统。
2. 简短说明：系统覆盖门诊、住院、药品、费用和统计分析。
3. 三类角色入口：管理员、医生、病人。
4. 核心模块入口：住院管理、费用管理、统计分析。
5. 核心指标卡片，例如科室数量、医生数量、病人档案数量、待缴账单数量。

## 5. 管理员工作台需求

管理员页面展示基础资料维护能力，应包含：

1. 科室信息：科室名称、位置、医生数、排班数。
2. 医生信息：姓名、工号、科室、职称、电话。
3. 药品库存：药品编码、名称、规格、库存、单价。
4. 可扩展病房和病床信息展示。
5. 表格搜索功能。

## 6. 医生工作台需求

医生页面展示医生端业务，应包含：

1. 医生排班：医生、科室、排班类型、时间、地点。
2. 挂号与接诊：病人、医生、科室、初诊/复诊、挂号时间、状态。
3. 门诊处方：病人、医生、诊断、金额、支付状态。
4. 住院巡诊入口或住院记录入口。

## 7. 病人中心需求

病人页面展示病人端业务，应包含：

1. 病人档案：姓名、病案号、性别、电话、当前状态。
2. 就诊记录：诊断、医生、费用、处方状态。
3. 费用明细：账单号、来源、金额、支付状态。
4. 前往缴费页面入口。

## 8. 住院管理需求

住院页面展示住院部业务，应包含：

1. 住院档案：住院号、病人、科室、主治医生、病床、住院状态。
2. 每日住院记录：病人、日期、病情、诊疗方案、费用。
3. 住院核心指标：住院档案数、在院病人数、可用床位数、住院记录数。

## 9. 费用管理需求

费用页面展示账单与缴费状态，应包含：

1. 账单列表：账单号、病人、来源、金额、状态。
2. 处方费用：病人、医生、诊断、金额、状态。
3. 状态只展示 `UNPAID` 和 `PAID`，与数据库设计保持一致。
4. 显示未支付账单合计。

## 10. 统计分析需求

统计页面展示课程要求中的统计查询能力，应包含：

1. 不同科室的排班情况。
2. 不同医生的工作量情况。
3. 病人的治疗情况概览。
4. 使用简单 CSS/JS 条形图展示统计结果，不引入图表库。

## 11. 通用交互需求

1. 顶部导航栏，支持页面跳转。
2. 当前页面导航高亮。
3. 表格支持关键词搜索。
4. 状态使用标签样式展示，例如 `PAID`、`UNPAID`、`ACTIVE`、`REGISTERED`。
5. 页面在常见屏幕宽度下可阅读，基础响应式布局。

## 12. 视觉风格需求

1. 医疗系统风格，主色建议使用蓝绿色。
2. 页面结构清晰，适合课程答辩展示。
3. 使用卡片、表格、状态标签和轻量统计图表。
4. 不做复杂动画，不使用外部图片和 CDN。
5. 优先保证信息完整、可读、易演示。

## 14. 后续可调整项

你可以修改本需求文档中的以下内容后再重新生成页面：

1. 页面数量。
2. 每个页面展示哪些模块。
3. 是否需要登录页。
4. 是否需要更像后台管理系统或更像展示大屏。
5. 是否需要表单新增/编辑弹窗。
6. 是否需要与后端接口对接。
7. 页面风格、颜色和布局。
